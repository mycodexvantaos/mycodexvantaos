## 🎯 量化交付標準

### 1. 代碼實現（數值化要求）

#### 完整性檢查清單
- ✅ 模塊文檔字符串：職責描述 + 用法示例
- ✅ 全公開函數的類型註解（支持情況下）
- ✅ 全公開函數的文檔字符串：參數→返回值→異常
- ✅ 關鍵邏輯（複雜度>3）的行內註釋
- ✅ 輸入驗證邏輯
- ✅ 異常捕獲與優雅降級
- ✅ 日誌記錄（關鍵路徑：啟動/請求開始結束/錯誤）

#### 複雜度與可維護性
- **圈複雜度**：單函數 ≤ 10（超過需重構）
- **函數長度**：單函數 ≤ 50 行（超過需分解）
- **類職責**：單一職責原則（SRP）
- **命名規範**：自說明性，避免單字母（除循環變量）

#### 依賴管理
- **核心功能**：零依賴或最小化（≤3個）
- **版本範圍**：`package>=1.0,<2.0`（兼容性優先）
- **衝突聲明**：標註已知不兼容的庫版本組合

### 2. 測試代碼（形式化要求）

#### 測試維度矩陣（必須覆蓋）
| 維度 | 正常 | 邊界 | 異常 | 並發 | 長時間 |
|------|:----:|:----:|:----:|:----:|:-----:|
| 覆蓋 | ✅ | ✅ | ✅ | ⚠️ | ⚠️ |

#### 覆蓋率指標
- 行覆蓋率 ≥ 90%（核心邏輯必須100%）
- 分支覆蓋率 ≥ 80%
- 條件覆蓋率 ≥ 75%

#### 測試隔離要求
- 單個測試無前置依賴
- 測試執行順序可任意
- 測試間無狀態共享
- 完整清理（setUp/tearDown）

### 3. 驗證報告（實證主義）

#### 正確性驗證
- [ ] CI/CD流水線通過標誌
- [ ] 測試運行截圖/日誌
- [ ] 需求追溯矩陣（代碼行→需求項）

#### 性能基準
- 響應時間：P95 < [閾值]ms
- 吞吐量：> [閾值] QPS
- 資源消耗：CPU < [閾值]%，內存 < [閾值]MB
- 與行業標準對比結果

#### 安全審計
- STRIDE分析結果（Spoofing/Tampering/Repudiation/Information Disclosure/Denial of Service/Elevation of Privilege）
- 依賴項安全掃描（0個Critical漏洞）
- 靜態掃描結果（Blocker/Critical → 0）
- 敏感信息檢查（無硬編碼密鑰、密碼等）

#### 資源管理驗證
- 連接池/連接數管理
- 文件句柄關閉驗證
- 內存洩漏測試結果
- 異常路徑下資源釋放確認

## 向後兼容性

### API演化策略

**添加新功能**：使用可選參數，不破壞現有簽名
```[語言]
# 舊簽名
def process(data: dict) -> str: ...

# 新簽名（兼容）
def process(data: dict, mode: str = "default") -> str: ...
```

### 棄用通知

若需破壞兼容性：
1. 當前版本添加 `@deprecated` 標記
2. 提供3個版本的棄用期
3. 提供明確的遷移指南
4. 在主版本變更時移除

```[語言]
@deprecated("使用 process_v2 替代，遷移指南見 ...")
def process_v1(data: dict) -> str:
    ...
```

在進入實現前，確認以下檢查清單：

- [ ] RSD已確認（無模糊詞彙）
- [ ] 所有假設已列舉
- [ ] 技術棧已選定
- [ ] 非功能性指標已量化
- [ ] 驗收標準已定義（Given-When-Then）
- [ ] 依賴和兼容性已評估
- [ ] 風險清單已生成

若有任何項未完成，反饋至對應階段，要求澄清/確認。

### 一次性交付原則

所有交付物一次性完成：

1. **代碼實現**：完整可運行
2. **測試套件**：包含所有維度用例
3. **驗證報告**：功能+性能+安全+資源
4. **文檔**：快速入門+API+使用說明
5. **配置文件**：依賴清單+環境要求

**禁止中途確認或等待**：除非發現無法解決的資訊缺口，否則持續到完成。

## 非功能性權衡優先級

當指標衝突時，按以下優先級處理（除非需求明確指定）：

1. **正確性（Correctness）**
   - 功能必須100%滿足需求
   - 測試通過率必須100%
   - 無安全漏洞（Critical/High級）

2. **可讀性/可維護性（Readability/Maintainability）**
   - 圈複雜度 ≤ 10
   - 自說明的命名
   - 完整的文檔字符串
   - 清晰的代碼組織

3. **安全性（Security）**
   - 必須阻斷常見漏洞
   - 依賴項安全（無Critical漏洞）
   - 敏感信息保護

4. **性能（Performance）**
   - 在滿足以上三點前提下追求最優
   - 標準化的優化方向
   - 可量化的性能基準
